Sample SQL Queries Used in Project

📝 Query 1: Data Cleaning & Deduplication -- Remove duplicate sales records and handle missing values WITH CleanedSales AS ( SELECT DISTINCT SaleID, CAST(SalesDate AS DATE) as Sale_Date, COALESCE(Amount, 0) as Amount, RegionID, ChannelID, ISNULL(CustomerID, 'GUEST') as CustomerID FROM RawSalesData WHERE Amount > 0 AND SalesDate IS NOT NULL ) INSERT INTO Sales_Cleaned SELECT * FROM CleanedSales;
📊 Query 2: Regional Sales Aggregation -- Calculate total sales by region with growth metrics SELECT r.State, r.Region, COUNT(DISTINCT s.SaleID) as Total_Orders, SUM(s.Amount) as Total_Sales, AVG(s.Amount) as Avg_Order_Value, ROUND(SUM(s.Amount) * 100.0 / (SELECT SUM(Amount) FROM Sales), 2) as Sales_Percentage FROM Sales s JOIN Regions r ON s.RegionID = r.RegionID WHERE YEAR(s.SalesDate) = 2024 GROUP BY r.State, r.Region ORDER BY Total_Sales DESC;
🏪 Query 3: Channel Performance Comparison -- Compare sales performance across e-commerce channels SELECT c.ChannelName, COUNT(s.SaleID) as Order_Count, SUM(s.Amount) as Total_Revenue, AVG(s.Amount) as Avg_Order_Value, COUNT(DISTINCT s.CustomerID) as Unique_Customers, ROUND(COUNT(s.SaleID) * 100.0 / (SELECT COUNT(*) FROM Sales), 2) as Market_Share FROM Sales s JOIN Channels c ON s.ChannelID = c.ChannelID GROUP BY c.ChannelName ORDER BY Total_Revenue DESC;
👥 Query 4: Customer Demographics Segmentation -- Segment customers by gender and analyze purchase behavior SELECT cust.Gender, COUNT(DISTINCT cust.CustomerID) as Total_Customers, COUNT(o.OrderID) as Total_Orders, SUM(s.Amount) as Total_Spending, AVG(s.Amount) as Avg_Spending, COUNT(CASE WHEN o.OrderStatus = 'Delivered' THEN 1 END) as Successful_Orders, ROUND(COUNT(CASE WHEN o.OrderStatus = 'Delivered' THEN 1 END) * 100.0 / COUNT(o.OrderID), 2) as Fulfillment_Rate FROM Customers cust LEFT JOIN Orders o ON cust.CustomerID = o.CustomerID LEFT JOIN Sales s ON o.OrderID = s.OrderID GROUP BY cust.Gender;
✅ Query 5: Fulfillment & Order Status Metrics -- Calculate fulfillment success rates and cancellation metrics SELECT o.OrderStatus, COUNT(o.OrderID) as Order_Count, ROUND(COUNT(o.OrderID) * 100.0 / (SELECT COUNT(*) FROM Orders), 2) as Status_Percentage, AVG(DATEDIFF(DAY, o.OrderDate, o.DeliveryDate)) as Avg_Delivery_Days, SUM(s.Amount) as Status_Revenue FROM Orders o LEFT JOIN Sales s ON o.OrderID = s.OrderID WHERE YEAR(o.OrderDate) = 2024 GROUP BY o.OrderStatus ORDER BY Order_Count DESC;

Backend Performance Optimization

Indexing Strategy

Implemented composite indexes on frequently queried columns:

  • Index on (RegionID, SalesDate)
  • Index on (ChannelID, Amount)
  • Index on (CustomerID, OrderStatus)

Partitioning

Sales table partitioned by year for improved query performance and maintenance.

  • Partition by YEAR(SalesDate)
  • Improves query speed by 40-50%
  • Easier data archival

Data Warehouse

Implemented star schema with fact and dimension tables.

  • Fact table: Sales_Fact
  • Dimensions: Time, Region, Channel
  • Pre-aggregated cubes for reporting

About The Project

The Fit & Fab Annual Report 2023 was developed to transform raw sales data into actionable business intelligence. By leveraging advanced data visualization techniques, this project provides a comprehensive overview of e-commerce performance across multiple dimensions.

Detailed Business Impact

In today's competitive e-commerce landscape, relying on intuition is no longer sufficient. This dashboard provides a strategic advantage by allowing stakeholders to:

  • Optimize Inventory Distribution: By identifying Maharashtra and Karnataka as top states, supply chain logistics can be prioritized to reduce shipping times.
  • Refine Marketing Spend: Knowing that Myntra is the dominant channel allows for targeted ad allocations, maximizing Return on Ad Spend (ROAS).
  • Tailor Product Offerings: Recognizing the strong sales contribution from women helps in curating future collections and promotional campaigns.
